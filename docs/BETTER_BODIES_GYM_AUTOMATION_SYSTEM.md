# 1) ONE-PAGE SYSTEM SUMMARY

## What it does
This system gives Better Bodies Gym one operating spine for leads, tours, members, payments, attendance, and follow-up tasks. It runs on Google Forms + Google Sheets + Gmail + WhatsApp Business (manual send or semi-automated send). It reduces drift by making every lead visible, assigning next actions, and alerting early when numbers dip.

Core outcomes:
- No lead should be unlogged.
- First response target is under 2 minutes during business hours.
- Every tour is tracked as show / no-show / close / lost.
- Payment failures are flagged on Day 1, Day 3, Day 7.
- Attendance drift is flagged at 7 and 14 days.
- Owner gets a Monday pulse report and critical alerts.

## What it refuses to do
- It does not replace your billing provider or banking system.
- It does not promise full WhatsApp bot automation (system still works with manual WhatsApp send).
- It does not run custom software development.
- It does not hide bad data; it forces clean inputs and simple statuses.

## Who runs what (Front desk vs Owner vs Founder A/B)
- Front Desk
  - Logs walk-ins and inbound messages using Lead Intake Form.
  - Sends WhatsApp follow-ups from daily task list.
  - Updates tour outcomes and payment notes.
  - Completes attendance correction if scanner/manual log missed.
- Owner
  - Handles escalations (high-value lead, repeated no-show, payment risk).
  - Reviews daily exception list (overdue follow-ups, payment Day 7, 14-day no-attendance).
  - Reviews Monday pulse report and sets one weekly focus action.
- Founder A (Operations)
  - Maintains sheet formulas, automations, and role permissions.
  - Tunes thresholds monthly.
- Founder B (Growth/Sales)
  - Owns scripts, offer wording, and follow-up quality checks.
  - Runs weekly sales coaching based on conversion metrics.

## Daily rhythm (10â€“20 min blocks)
- 06:00 opening block (10 min)
  - Check new leads since last shift.
  - Send immediate responses for any unsent leads.
  - Confirm same-day tour bookings.
- 12:00 midday block (10 min)
  - Clear overdue follow-up tasks.
  - Mark show/no-show for morning tours.
- 17:00 evening block (15â€“20 min)
  - Send Day 1/3/7 follow-ups.
  - Resolve payment reminders due today.
  - Trigger owner escalation list.

## Weekly rhythm (Monday report)
- Monday 08:00â€“08:30
  - Auto-generated weekly report sent to owner by email.
  - Owner posts summary to management WhatsApp group.
  - Team picks top 2 fixes for the week (example: improve Day 0 response, reduce no-shows).

# 3) WORKFLOWS (TRIGGERS â†’ ACTIONS â†’ EXCEPTIONS â†’ STOP RULES)

## A) Lead capture (all sources â†’ one sheet)
Trigger
- New inquiry from walk-in, Instagram DM, Facebook message, WhatsApp message.

Actions
1. Front desk opens `Lead Intake Form` (mobile shortcut).
2. Completes minimum fields in under 45 seconds: name, mobile, source, interest.
3. Form writes to `Leads` tab and auto-generates LeadID.
4. Automation sends internal alert email: `New Lead: <LeadID> needs reply now`.
5. Front desk sends immediate WhatsApp template reply.
6. `Staff Tasks` auto-creates Day 0 task due immediately.

Exceptions
- Duplicate check: if same mobile exists in last 60 days with status not `Lost`, mark new row as `Possible Duplicate` and assign to owner review.
- After-hours inquiry: set `NextActionDate` to next opening time (06:00 next open day), send after-hours acknowledgement.

Stop rules
- Stop capture workflow when LeadStatus becomes `Booked Visit`, `Won`, or `Lost`.

Response-time rule
- KPI: first outbound response in <2 minutes during operating hours.
- Alert if response exceeds 5 minutes for any lead during open hours.

## B) Lead follow-up sequence (sales)
Trigger
- LeadStatus = `New` or `Contacted` and no booking yet.

Actions
- Day 0: immediate response + offer 2 visit times.
- Day 1: follow-up reminder with clear CTA to book visit.
- Day 3: value message + simple options (`book`, `questions`, `not now`).
- Day 7: final polite check-in + archive option.
- Every send logged in `Staff Tasks` with outcome.

Book a visit script path
1. Ask preferred day/time.
2. Confirm slot.
3. Create `Booking` row.
4. Send confirmation template.

Come in now path
1. Confirm gym is open now.
2. Ask ETA.
3. Mark booking as `Walk-in Expected` (or immediate booking at current time).
4. Alert front desk.

Exceptions
- Lead asks price only: send membership options template with promo placeholder.
- Lead says "not now": set status `Paused`, next action +14 days.
- No WhatsApp delivery: switch to email if available.

Stop rules
- Stop sequence if lead books visit, joins, or opts out.
- Escalate to owner on Day 3 if high intent but no booking (e.g., asked payment questions twice).

Escalation
- Owner takes over when:
  - Lead has not replied after Day 3 but was previously active.
  - Lead requests manager callback.
  - Corporate/family package inquiry.

## C) Tour/visit handling (show-up â†’ close)
Trigger
- Booking created in `Tours/Visits`.

Actions
1. Send booking confirmation immediately.
2. Send reminder 3 hours before visit.
3. On arrival, set status `Completed` and record outcome.
4. Front desk offers two plans and joining fee (with promo logic).
5. If member signs: create Member row + initial Payment rows.

Missed visit flow
- If no-show at +20 minutes:
  - Mark `No-show`.
  - Send missed appointment template.
  - Auto-create follow-up task for next day.

If they donâ€™t buy
- Outcome `Thinking`.
- Schedule follow-up at +1 day and +3 days.

Exceptions
- Late arrival >30 min and no slot: reschedule immediately.
- Price objection: offer to hold promo end date only if promo active.

Stop rules
- Stop when outcome is `Joined` or `Not Interested` with confirmed reason.

## D) New member onboarding
Trigger
- New `Member` row created.

Actions
1. Send welcome message within 10 minutes.
2. Send gym rules + operating hours + billing explainer.
3. Create first-week check-in task (Day 3 or Day 4).
4. Monitor first 14 days attendance (auto flag if no attendance for 3+ days in first week).

Exceptions
- Member joins after staffed hours: send welcome at 06:00 next day.
- Missing email/phone: front desk must correct profile before end of day.

Stop rules
- End onboarding workflow after Day 14 and at least 2 attendance logs.
- If attendance under target, hand off to churn prevention workflow.

## E) Payments monitoring (contracts + joining fee + promos)
Trigger
- Payment row reaches DueDate and status not `Paid`.

Actions
1. Day 1 overdue: reminder message 1 + task log.
2. Day 3 overdue: reminder message 2 + owner copied on alert.
3. Day 7 overdue: reminder message 3 + escalation call task.
4. Optional: set MemberStatus to `At Risk` or `Suspended Pending Payment` (if gym policy chosen).

Plan support
- 12-month contract: R249/month.
- 24-month contract: R178/month.
- Joining fee default: R250.
- Promo joining fee: dynamic from Membership Plans tab.

Exceptions
- Approved waiver/promo: set status `Waived` with reason and approver.
- Bank delay proof provided: extend due date by 2 business days and log note.

Stop rules
- Stop reminders when PaymentStatus changes to `Paid`, `Waived`, or `Refunded`.

## F) Churn prevention (attendance drift)
Trigger
- No attendance for 7 days OR 14 days.

Actions
- At 7 days:
  - Create staff task: WhatsApp check-in + optional voice note.
  - Ask for barrier: time, motivation, injury, finance.
- At 14 days:
  - Owner escalation call.
  - Offer retention options: freeze, downgrade/upgrade, accountability check-in plan.

Exceptions
- Member already `Frozen`: suppress 7/14-day alerts.
- Medical leave note: snooze risk tasks until return date.

Stop rules
- Stop churn sequence when member attends again OR plan formally frozen/cancelled.

## G) BUSINESS PULSE ENGINE (alerts + weekly report)
Inputs (minimum)
- Leads count (new per week)
- Contact under 2 min rate
- Booked visits count
- Visit show-up rate
- Lead-to-member conversion rate
- Weekly collected revenue
- Payment failures count
- Active members count
- Attendance sessions per week
- 7-day and 14-day at-risk members

Baselines
- Compare each metric to:
  - Last week.
  - Rolling 4-week average.

Threshold defaults
- Revenue drop alert: 20% vs 4-week average.
- Leads drop: 25%.
- Conversion drop: 25%.
- Attendance drop: 15%.
- Payment failures: >5/week OR +30% vs baseline.

Outputs
- Instant alert format (WhatsApp/email):
  - `ALERT | Better Bodies Westonaria | Metric: Payment Failures | This week: 8 | 4-week avg: 5 | Change: +60% | Action today: call all D7 overdue members by 16:00.`
- Weekly report format (copy/paste):
  1. Leads: X (vs LW, vs 4W avg)
  2. Response <2 min: X%
  3. Tours booked: X, show-up: X%
  4. New members: X, conversion: X%
  5. Revenue collected: RX
  6. Payment failures: X
  7. At-risk members 7d/14d: X / Y
  8. Top 3 issues + assigned owner + due date

# 5) TOOL STACK (2 OPTIONS)

Components
- Google Form: Lead Intake Form.
- Google Sheets: all 7 tables + dashboard tab.
- Gmail: alerts and Monday report.
- Google Apps Script: ID generation, trigger-based task creation, weekly report email.
- WhatsApp Business app: staff sends templates manually.

Why chosen
- Lowest cost.
- Works tomorrow with existing tools.
- Minimal failure points.

Setup complexity
- Low to medium (1â€“2 days).

What can break and detection

## Option 2: More automated (Airtable/CRM + Make/Zapier + official messaging if needed)
Components
- Airtable or HubSpot Free for lead/member pipeline.
- Make or Zapier for workflow automation.
- Gmail for fallback and reports.
- WhatsApp via official provider (if approved) or manual send fallback.

Why chosen
- Better audit trail, cleaner task boards, easier scaling to multiple gyms.
- More automation of reminders and escalations.

Setup complexity
- Medium (3â€“7 days).

What can break and detection
- Automation scenario paused/error â†’ detect with daily error digest email.
- API/message provider outages â†’ fallback to manual WhatsApp task queue.

# 7) OPERATING CHECKLISTS (NO THINKING REQUIRED)

## Front desk daily checklist
1. Open dashboard at shift start.
2. Process all `New Leads` older than 2 minutes.
3. Send due follow-ups (Day 1/3/7).
4. Confirm todayâ€™s bookings.
5. Mark tour outcomes immediately after each visit.

## Owner daily checklist

   - Overdue lead replies
   - Payment Day 7
   - 14-day no-attendance
2. Handle escalations and approvals (waivers/freezes).

## Weekly checklist (Monday pulse review)
1. Read weekly pulse email.
2. Compare against thresholds.
3. Pick top 3 issues and assign owner + deadline.
4. Review conversion by source.
5. Review payment failure reasons and fix root cause.

## Monthly checklist (threshold tuning + cleanup)
1. Audit data quality (missing phone, stale statuses, duplicate leads).
2. Recalculate 4-week baselines.
3. Tune thresholds only if repeated false positives.
4. Archive closed/lost items older than 90 days.
5. Confirm access permissions and remove inactive staff.

## Kill list rule (when overwhelmed)
Cut in this order (never cut lead capture or payment reminders):
1. Pause Day 7 sales follow-up first.
2. Pause non-critical dashboard formatting updates.
3. Pause optional growth experiments.
4. Keep only: lead capture, Day 0/1 follow-up, booking handling, payment Day 1/3/7, 14-day churn alerts.
